# 项目接口与脚本使用说明

本文档用于分发给小组成员，说明算法实验代码应输出什么格式，以及如何使用统一评估和单图可视化脚本。

## 1. 运行环境

统一使用 Python 实现，需要用到的库：

| 库 | 版本 | 用途 |
|---|---:|---|
| OpenCV / `cv2` | 4.10.0 | 图像读写、颜色空间转换、边缘检测、形态学、连通域、分水岭 |
| NumPy | 1.26.4 | 数组计算、投影统计、参数搜索 |
| Pandas | 2.2.3 | 标注表、结果表和汇总表处理 |
| Matplotlib | 3.9.2 | 实验曲线、统计图、可视化拼图 |
| scikit-image | 0.25.2 | 必要时辅助实现区域、形态学和测量功能 |

## 2. 关键目录

```text
dataset/                         原始 BMP 图像和 LabelMe JSON 标注
annotations/plate_char_annotations.csv
                                 人工标注真值表
src/convert_labelme_to_csv.py     LabelMe JSON 转真值 CSV
src/evaluate.py                   统一评估程序
src/visualize_single.py           单图四宫格可视化程序
results/                          算法预测 CSV 和评估结果
outputs/                          可视化图片输出
```

当前数据集共 30 张图像，每张车牌均为 7 个字符。真值 CSV 中已经包含 `plate` 框和 `char_1` 到 `char_7` 字符框。

## 3. 算法预测 CSV 格式

每个算法需要输出一个预测 CSV。推荐路径示例：

```text
results/<method_name>_predictions.csv
```

预测 CSV 必需列如下：

| 列名 | 是否必需 | 说明 |
|---|---|---|
| `image_name` | 必需 | 图像文件名，必须与 `dataset/` 中 BMP 文件名一致 |
| `method` | 必需 | 方法名称，例如 `edge_morph_projection` |
| `plate_bbox_pred` | 必需 | JSON 字符串，预测车牌框 `[x, y, w, h]` |
| `char_bboxes_pred` | 必需 | JSON 字符串，预测字符框列表 `[[x, y, w, h], ...]` |
| `params` | 可选 | 参数组合，建议保存为 JSON 字符串 |
| `runtime_ms` | 可选 | 单张图运行时间，单位毫秒 |
| `status` | 可选 | `success`、`plate_not_found`、`char_failed`、`exception` 等 |
| `failure_reason` | 可选 | 失败原因说明 |
| `binary_path` | 可选 | 字符二值图路径，可作为算法调试字段保留；统一单图可视化默认不再使用该字段 |
| `foreground_path` | 可选 | 字符前景图路径，可作为算法调试字段保留；统一单图可视化默认不再使用该字段 |

坐标必须使用原图坐标，不要使用车牌 ROI 内的局部坐标。矩形框格式统一为 OpenCV 风格：

```text
[x, y, w, h]
```

其中 `(x, y)` 是左上角，`w` 是宽度，`h` 是高度。

预测 CSV 示例：

```csv
image_name,method,plate_bbox_pred,char_bboxes_pred,runtime_ms,status,failure_reason
收费站0809.bmp,edge_morph_projection,"[314, 351, 180, 66]","[[322, 360, 18, 45], [344, 360, 20, 45], [368, 360, 22, 45], [393, 360, 18, 45], [416, 360, 18, 45], [438, 360, 20, 45], [462, 360, 21, 45]]",35.2,success,
```

空预测可以写为空字符串、`[]` 或 `null`。例如未定位到车牌时：

```csv
image_name,method,plate_bbox_pred,char_bboxes_pred,status,failure_reason
收费站0809.bmp,edge_morph_projection,"","[]",plate_not_found,edge candidates filtered out
```

## 4. 统一评估脚本

评估脚本读取人工标注真值 CSV 和算法预测 CSV，输出逐图指标、方法汇总指标和失败类型统计。

基本命令：

```powershell
conda run -n newd2l python src\evaluate.py --gt annotations\plate_char_annotations.csv --pred results\<method_name>_predictions.csv --out-dir results\eval_<method_name>
```

输出文件：

```text
results/eval_<method_name>/per_image_metrics.csv
results/eval_<method_name>/method_summary.csv
results/eval_<method_name>/failure_summary.csv
```

主要指标：

| 指标 | 说明 |
|---|---|
| `plate_iou` | 预测车牌框与人工车牌框的 IoU |
| `plate_center_error` | 预测框中心与人工框中心的欧氏距离 |
| `plate_success_50` | `plate_iou >= 0.5` |
| `plate_success_70` | `plate_iou >= 0.7` |
| `char_count_pred` | 预测字符框数量 |
| `char_count_correct` | 预测字符数量是否等于 7 |
| `char_mean_iou` | 7 个预测字符框与 7 个真值字符框的平均 IoU |
| `char_boundary_error` | 7 个字符左右边界平均误差 |
| `over_split` | 预测字符数大于 7 |
| `under_split` | 预测字符数小于 7 |
| `end_to_end_success` | `plate_iou >= 0.5` 且预测字符数为 7 且 `char_mean_iou >= 0.45` |

字符框评估时，程序会把预测字符框按 `x` 坐标从左到右排序，再与 `char_1` 到 `char_7` 匹配。

## 5. 单图可视化脚本

单图可视化脚本用于生成实验计划 7.1 中的四宫格图：

1. 原图 + 人工车牌框 + 预测车牌框。
2. 车牌 ROI。
3. 人工字符框面板：只显示人工标注字符框。
4. 预测字符框面板：只显示算法预测字符框。

GT 与 Pred 字符框分开显示，避免两组框和文字标签互相遮挡。字符标签位于图像上方的独立标签条内，不覆盖车牌字符主体。

基本命令：

```powershell
conda run -n newd2l python src\visualize_single.py --gt annotations\plate_char_annotations.csv --pred results\<method_name>_predictions.csv --image-dir dataset --out-dir outputs\single_vis_<method_name>
```

如果已经运行过评估脚本，建议把逐图指标也传给可视化脚本：

```powershell
conda run -n newd2l python src\visualize_single.py --gt annotations\plate_char_annotations.csv --pred results\<method_name>_predictions.csv --image-dir dataset --out-dir outputs\single_vis_<method_name> --metrics results\eval_<method_name>\per_image_metrics.csv
```

常用筛选：

```powershell
# 只生成前 5 张，快速检查格式
conda run -n newd2l python src\visualize_single.py --gt annotations\plate_char_annotations.csv --pred results\<method_name>_predictions.csv --image-dir dataset --out-dir outputs\single_vis_preview --limit 5

# 只看某个方法
conda run -n newd2l python src\visualize_single.py --gt annotations\plate_char_annotations.csv --pred results\all_predictions.csv --image-dir dataset --out-dir outputs\single_vis_edge --method edge_morph_projection

# 只生成失败样本，必须提供 metrics
conda run -n newd2l python src\visualize_single.py --gt annotations\plate_char_annotations.csv --pred results\<method_name>_predictions.csv --image-dir dataset --out-dir outputs\single_vis_failures --metrics results\eval_<method_name>\per_image_metrics.csv --only-failures
```

颜色约定：

| 元素 | 颜色 |
|---|---|
| 人工车牌框 | 绿色 |
| 预测车牌框 | 红色 |
| 人工字符框 | 绿色，标签为 `G1-G7` |
| 预测字符框 | 黄色，标签为 `P1-P7` |
| 失败或异常提示 | 红色文字 |

## 6. 提交给组内汇总时需要提供的文件

每位同学完成自己的算法后，至少提交：

1. 算法代码。
2. 预测 CSV，例如 `results/<method_name>_predictions.csv`。
3. 评估输出目录，例如 `results/eval_<method_name>/`。
4. 单图可视化目录，例如 `outputs/single_vis_<method_name>/`。
5. 简短说明：方法名、主要参数、典型失败原因。

预测 CSV 是最重要的接口文件。只提交截图无法参与统一评估和横向对比。

## 7. 快速自测流程

项目中已经包含 smoke test 输入，可用于确认评估和可视化脚本是否正常：

```powershell
conda run -n newd2l python src\evaluate.py --gt annotations\plate_char_annotations.csv --pred results\eval_smoke_inputs\perfect.csv --out-dir results\eval_smoke_perfect

conda run -n newd2l python src\visualize_single.py --gt annotations\plate_char_annotations.csv --pred results\eval_smoke_inputs\perfect.csv --image-dir dataset --out-dir outputs\single_vis_smoke_perfect --metrics results\eval_smoke_perfect\per_image_metrics.csv --limit 2
```

完美预测的 `method_summary.csv` 中应看到：

```text
plate_success_rate_50 = 1.0
char_count_accuracy = 1.0
avg_char_iou = 1.0
end_to_end_success_rate = 1.0
```

## 8. 常见问题

1. **Pylance 提示 pandas/cv2 找不到**  
   通常是 VS Code 没选中 `newd2l` 解释器。运行脚本时以 `conda run -n newd2l` 为准。

2. **评估脚本提示缺少列**  
   检查预测 CSV 是否包含 `image_name`、`method`、`plate_bbox_pred`、`char_bboxes_pred`。

3. **字符 IoU 是空值**  
   当预测字符数量不是 7 时，程序不会强行计算一一匹配 IoU，而是记录过分割或欠分割。

4. **可视化第三格不是二值图**  
   这是新版设计：第三格固定显示人工字符框，用于和第四格预测字符框进行直观对比。`binary_path` 和 `foreground_path` 仅作为算法调试字段保留。

5. **中文路径图片读写失败**  
   当前脚本使用 `cv2.imdecode(np.fromfile(...))` 和 `cv2.imencode(...).tofile(...)`，已针对 Windows 中文路径处理。如果仍失败，优先检查文件是否存在。
