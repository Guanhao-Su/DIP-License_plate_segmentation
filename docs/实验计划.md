# 数字图像处理大作业：车牌字符分割实验计划

## 1. 任务目标与总体思路

本次大作业的目标是基于课程第九章“图像分割”中的传统图像处理方法，对给定车辆图像中的车牌字符进行分割。老师给出的提示强调：车牌区域通常具有丰富边缘，形态学、区域增长等课内方法均可使用。因此，本组实验不采用深度学习方法，而是围绕阈值分割、边缘检测、形态学处理、区域增长、分水岭等方法设计可解释、可比较的实验流程。

数据集位于 `dataset/` 文件夹，共 30 张 BMP 图像，图像尺寸均为 `768 x 576`，包含收费站白天图像、夜间或强反光图像，以及蓝牌、黄牌、白牌等不同车牌外观。由于原始数据集没有附带标注文件，本实验需要先建立轻量人工标注，用于后续定量评价。

总体流程采用“两阶段”设计：

1. **车牌定位**：从整幅车辆图像中检测车牌区域，输出车牌外接矩形框。
2. **字符分割**：在车牌 ROI 中提取字符前景，并切分出每个字符的外接矩形框。

这样设计的原因是：老师作业标题强调“车牌分割”，示例中最终结果是字符框；但实际数据是整车图像，若不先定位车牌，字符分割难以直接进行。因此实验中既评价车牌定位，也评价字符分割，并最终评价端到端效果。

## 3. 统一输入输出规范

为保证三名成员实现的算法能够横向比较，所有算法必须遵守统一接口。建议每种算法都实现为一个独立函数或脚本，但输入输出字段保持一致。

### 3.1 输入

每个算法至少接收：

| 字段 | 类型 | 说明 |
|---|---|---|
| `image_path` | string | 原始 BMP 图像路径 |
| `plate_bbox` | list 或空 | 可选人工车牌框，格式为 `[x, y, w, h]` |
| `params` | dict | 当前实验参数组合 |

其中 `plate_bbox` 用于两类实验：

1. **人工 ROI 字符分割实验**：直接使用人工车牌框，只比较字符分割方法。
2. **自动端到端实验**：不提供人工车牌框，由算法先定位车牌，再分割字符。

### 3.2 输出

每种算法输出统一结果，统一保存为预测 CSV 中的一行，供 `src/evaluate.py` 和 `src/visualize_single.py` 读取：

| 字段 | 类型 | 说明 |
|---|---|---|
| `image_name` | string | 图像文件名 |
| `method` | string | 方法名称，如 `edge_morph_plate`、`otsu_projection_chars` |
| `params` | string | 可选，参数组合，建议保存为 JSON 字符串 |
| `plate_bbox_pred` | JSON 字符串或空 | 预测车牌框 `[x, y, w, h]` |
| `char_bboxes_pred` | JSON 字符串或空 | 预测字符框列表 `[[x, y, w, h], ...]`，坐标必须使用原图坐标 |
| `runtime_ms` | float | 单张图运行时间，单位毫秒 |
| `status` | string | `success`、`plate_not_found`、`char_failed`、`exception` 等 |
| `failure_reason` | string | 失败原因，如强反光、车牌过暗、字符粘连、定位偏移 |
| `binary_path` | string | 可选，算法输出的字符二值图路径，可作为算法调试字段保留；统一单图可视化默认不再使用该字段 |
| `foreground_path` | string | 可选，算法输出的字符前景图路径，可作为算法调试字段保留；统一单图可视化默认不再使用该字段 |
| `vis_path` | string | 可选，算法自带可视化路径；统一可视化脚本不依赖该字段 |

坐标统一采用 OpenCV 风格：左上角为 `(0, 0)`，`x` 向右，`y` 向下，矩形框为 `[x, y, w, h]`。

## 4. 标注工具与人工标注方案

由于数据量只有 30 张，每张图进行较细致的人工标注是可以接受的。本实验建议不只标注车牌框，还尽量标注每个字符框。这样既能评价车牌定位，也能评价字符分割，不需要只依赖人工主观判断。

### 4.1 标注工具选择

本机已确认可用 `labelme 5.6.0` 和 `labelImg`。建议采用：

| 工具 | 推荐程度 | 适合内容 | 说明 |
|---|---|---|---|
| LabelMe | 推荐作为主工具 | 车牌框、字符框、必要时的多边形精细轮廓 | 支持矩形、多边形、点等多种形状，导出 JSON，后续可写脚本转换为 CSV |
| LabelImg | 可作为备选 | 只画矩形框 | 操作更简单，但主要面向目标检测矩形框，不如 LabelMe 灵活 |

本实验主任务是分割字符，但评价时最关键的是字符位置和字符数量，因此推荐采用 **LabelMe 的 rectangle 标注**：

1. 对整张图画一个车牌矩形框，标签为 `plate`。
2. 对车牌内 7 个字符分别画矩形框，标签为 `char_1`、`char_2`、...、`char_7`，从左到右编号。
3. 如果字符存在少量遮挡或反光干扰，仍按正常 `char_i` 标注，后续只在 `notes` 中说明。
4. 每张图片保存一个 LabelMe JSON 文件，后续统一转换为 `annotations/plate_char_annotations.csv`。

LabelMe 启动命令建议为：

```powershell
conda run -n labelme labelme dataset
```

如果该命令在 PowerShell 中不能直接打开图形界面，可先激活环境后运行：

```powershell

```

## 5. 算法设计与参数对比

本实验分为“车牌定位方法”和“字符分割方法”两类，但两者不是完全独立的任务。在端到端流程中，字符分割依赖车牌定位结果；如果车牌框偏移，后续字符分割即使算法本身正确，也会受到影响。

为了保证小组进度可以并行推进，本实验采用 **人工 ROI 解耦 + 端到端组合验证** 的策略：

1. **人工 ROI 字符分割实验**：直接使用人工标注的车牌框裁剪 ROI。这样每位成员都可以独立开发字符分割方法，不必等待车牌定位算法完成。
2. **自动车牌定位实验**：只评价定位算法本身，输出预测车牌框，并与人工车牌框计算 IoU。
3. **端到端组合实验**：将每个人的定位方法和字符分割方法组合运行，观察定位误差对字符分割的影响。

因此，分工时不建议把“全部车牌定位”交给一个人、“全部字符分割”交给另一个人。更合理的方式是：每个人负责一个相对完整的算法包，算法包内包含一种车牌定位思路和一种字符分割思路，最后三个人进行横向比较。

### 5.1 车牌定位方法一：边缘密集度 + 形态学

理论依据：车牌区域含有密集字符边缘，边缘方向以垂直笔画和矩形边框为主。课件中边缘检测和形态学后处理可结合使用。

基本流程：

1. 灰度化。
2. 高斯滤波，降低噪声和反光小区域影响。
3. 使用 Sobel 或 Canny 提取边缘。
4. 使用横向闭运算连接字符边缘，使车牌区域形成较大连通块。
5. 连通域分析，筛选符合车牌长宽比、面积、矩形度的候选框。
6. 对候选框进行评分，选出最可能的车牌区域。

建议比较参数：

| 参数 | 候选值 | 影响 |
|---|---|---|
| 高斯核大小 | `3, 5, 7` | 核越大越抑制噪声，但可能模糊字符边缘 |
| Canny 低阈值 | `50, 80, 100` | 影响弱边缘保留程度 |
| Canny 高阈值 | `120, 160, 200` | 影响强边缘判定 |
| Sobel 方向 | `x`、`y`、`x+y` | 车牌字符垂直边缘通常对 `x` 方向梯度更敏感 |
| 闭运算核 | `(15,3), (25,5), (35,5)` | 横向连接字符，核过大会连接车身边缘 |
| 车牌长宽比 | `2.0-5.5` | 用于过滤非车牌区域 |
| 最小面积 | `500, 1000, 2000` | 过滤小噪声连通域 |

预期优点：对蓝牌字符清晰、边缘丰富的图像效果较好。

可能失败：强反光车灯、车身格栅、保险杠纹理也会产生密集边缘，容易误检。

### 5.2 车牌定位方法二：HSV 颜色阈值 + 连通域筛选

理论依据：车牌区域具有明显颜色特征，例如蓝牌、黄牌、白牌背景与车身存在颜色差异。将 RGB 转为 HSV 后，可以更稳定地利用色调和饱和度进行分割。

基本流程：

1. RGB/BGR 转 HSV。
2. 分别设置蓝牌、黄牌、白牌的颜色阈值。
3. 得到颜色掩膜后进行开运算去噪、闭运算填洞。
4. 连通域分析，筛选长宽比和面积符合车牌的候选框。
5. 多颜色候选中选择评分最高者。

建议比较参数：

| 车牌类型 | HSV 阈值方向 | 调参重点 |
|---|---|---|
| 蓝牌 | H 约在蓝色范围，S 较高 | 适应曝光变化，避免车身蓝色干扰 |
| 黄牌 | H 约在黄色范围，S 较高 | 夜间偏白或偏橙时阈值需放宽 |
| 白牌 | S 较低，V 较高 | 容易与车灯、反光区域混淆，需结合矩形度和位置 |

建议比较参数：

| 参数 | 候选值 | 影响 |
|---|---|---|
| 蓝牌 H 范围 | 根据样本统计设 2-3 组 | 控制蓝牌召回率 |
| 黄牌 H 范围 | 根据样本统计设 2-3 组 | 控制黄牌召回率 |
| 白牌 V 下限 | `160, 180, 200` | 控制白牌与反光区域区分 |
| 开运算核 | `(3,3), (5,5)` | 去除小噪声 |
| 闭运算核 | `(9,3), (15,5)` | 连接车牌背景区域 |
| 面积阈值 | `500, 1000, 2000` | 过滤小区域 |

预期优点：对颜色明显的车牌定位较快，解释性强。

可能失败：强反光、过曝、白牌与车灯区域接近时容易混淆。

### 5.3 车牌定位方法三：边缘与颜色融合候选评分

为提高鲁棒性，可将方法一和方法二产生的候选框合并，再统一评分。

候选框评分建议：

```text
score = 0.30 * aspect_score
      + 0.25 * edge_density_score
      + 0.20 * color_ratio_score
      + 0.15 * rectangularity_score
      + 0.10 * position_score
```

其中：

- `aspect_score`：长宽比越接近常见车牌比例越高。
- `edge_density_score`：候选区域内部边缘密度适中且较集中时得分高。
- `color_ratio_score`：蓝/黄/白车牌颜色像素占比高时得分高。
- `rectangularity_score`：连通域面积与外接矩形面积之比。
- `position_score`：车牌通常位于车辆前脸中下部，可作为弱先验，不可权重过高。

此方法作为最终端到端流程的推荐车牌定位方案。

### 5.4 字符分割方法一：Otsu / 自适应阈值 + 垂直投影

该方法由本人重点负责，作为基础字符分割方案。

理论依据：课件中指出大津法适合文本、车牌等目标与背景相对紧凑的图像；车牌字符在二值图中沿水平方向排列，因此可以利用垂直投影寻找字符间空隙。

基本流程：

1. 根据车牌框裁剪 ROI。
2. 对 ROI 做尺寸归一化，例如统一高度为 80 像素。
3. 灰度化，并根据车牌颜色判断是否需要反色，使字符为白色、背景为黑色。
4. 使用 Otsu 或自适应阈值得到二值图。
5. 使用开运算去除孤立噪声，闭运算修复字符断裂。
6. 计算每一列的前景像素数量，得到垂直投影曲线。
7. 平滑投影曲线，寻找低谷作为字符分割线。
8. 输出每个字符外接框。

建议比较参数：

| 参数 | 候选值 | 影响 |
|---|---|---|
| 二值化方法 | `Otsu`, `adaptive_mean`, `adaptive_gaussian` | 不同光照条件下稳定性不同 |
| 自适应阈值 block size | `15, 21, 31, 41` | 越大越接近全局阈值 |
| 自适应阈值 C | `-5, 0, 5, 10` | 控制前景面积 |
| ROI 归一化高度 | `60, 80, 100` | 影响后续形态学核尺度 |
| 开运算核 | `(2,2), (3,3)` | 去除噪声，过大会损伤细字符 |
| 闭运算核 | `(2,2), (3,3), (1,3)` | 修复断笔，过大会导致粘连 |
| 投影平滑窗口 | `3, 5, 7` | 影响切分线稳定性 |
| 最小字符宽度 | `5%, 8%, 10% ROI宽度` | 过滤噪声小块 |

预期优点：实现简单、速度快、适合字符清楚且未严重粘连的车牌。

可能失败：字符粘连、反光遮挡、车牌倾斜、背景与字符对比不足时，投影低谷不明显。

### 5.5 字符分割方法二：连通域分析

理论依据：二值图中每个字符通常对应一个或多个连通域。通过面积、高度、宽度、长宽比和位置可筛选字符区域。

基本流程：

1. 对车牌 ROI 二值化。
2. 使用形态学操作去除噪声并适当修补字符。
3. 提取连通域。
4. 过滤过小、过大、过扁、过高的区域。
5. 对字符候选框按 x 坐标排序。
6. 若一个字符被分裂为多个连通域，可根据水平距离和高度重叠进行合并。

建议比较参数：

| 参数 | 候选值 | 影响 |
|---|---|---|
| 最小面积比例 | `0.1%, 0.3%, 0.5% ROI面积` | 过滤噪声 |
| 字符高度比例 | `40%-95% ROI高度` | 过滤车牌边框和小噪声 |
| 字符宽度比例 | `3%-25% ROI宽度` | 过滤异常连通域 |
| 合并距离阈值 | `2, 4, 6` 像素 | 合并断裂字符 |

预期优点：对字符分离明显的图像效果直观，便于解释。

可能失败：省份简称、字母和数字存在内部断裂时，单字符可能被拆成多个连通域；粘连字符可能被当作一个连通域。

### 5.6 字符分割方法三：区域增长

理论依据：区域增长利用像素空间邻接关系和灰度/颜色相似性，将满足准则的相邻像素合并为同一区域。课件中强调区域增长需要解决种子选择、生长准则和停止条件三个问题。

本实验中的两种用途：

1. **车牌 ROI 细化**：从颜色阈值得到的候选车牌区域中选择种子，向周围增长，得到更完整的车牌背景区域。
2. **字符前景提取**：在二值化或边缘图的基础上选择字符种子，增长得到字符区域。

推荐优先使用第二种用途，因为作业最终关注字符分割。

基本流程：

1. 在车牌 ROI 内先进行粗二值化。
2. 选择高置信字符像素作为种子，例如二值图中面积较小但高度合理的连通域中心。
3. 使用 4 邻接或 8 邻接进行生长。
4. 生长准则可选灰度差、HSV 差或与当前区域均值的差。
5. 当没有新像素满足条件或区域面积超过上限时停止。
6. 对增长区域再做连通域分析，输出字符框。

建议比较参数：

| 参数 | 候选值 | 影响 |
|---|---|---|
| 邻接方式 | `4-neighbor`, `8-neighbor` | 8 邻接更容易连接斜向笔画，也更容易扩散 |
| 灰度差阈值 | `10, 15, 20, 30` | 控制生长范围 |
| HSV 距离阈值 | `15, 25, 35` | 对彩色 ROI 更稳定 |
| 种子选择方式 | 连通域中心、投影峰值列、人工高置信区域 | 决定区域增长起点 |
| 面积上限 | `20%, 30%, 40% ROI面积` | 防止扩散到背景 |

预期优点：能利用空间连续性，改善局部断裂字符。

可能失败：种子选错会导致错误区域增长；阈值过大时容易扩散到车牌背景或边框。

### 5.7 字符分割方法四：标记控制分水岭

理论依据：分水岭能基于灰度地形将图像划分为不同区域，但普通分水岭容易过分割。课件中提到可通过高斯平滑或标记控制减少过分割，因此本实验采用标记控制分水岭处理粘连字符。

基本流程：

1. 对车牌 ROI 二值化，使字符为前景。
2. 对前景图进行距离变换。
3. 取距离变换中的局部峰值作为字符内部 marker。
4. 将背景区域作为 background marker。
5. 在梯度图或反距离图上运行 watershed。
6. 将分水岭结果转换为字符区域。
7. 过滤异常区域并输出字符框。

建议比较参数：

| 参数 | 候选值 | 影响 |
|---|---|---|
| 距离变换阈值 | `0.3, 0.4, 0.5, 0.6 * max_distance` | 控制 marker 数量 |
| marker 最小距离 | `3, 5, 7` 像素 | 控制是否将粘连字符分开 |
| 梯度图来源 | Sobel、形态学梯度、反距离图 | 影响分割边界 |
| 前景闭运算核 | `(2,2), (3,3)` | 影响粘连程度 |

预期优点：适合处理粘连字符，是区域方法和形态学分割的综合应用。

可能失败：marker 数量不准确会导致过分割或欠分割；低质量二值图会明显影响结果。

## 6. 评价指标

本实验同时使用定量评价和定性评价。定量评价保证不同算法可横向比较，定性评价用于解释传统方法在复杂场景下的失败原因。

### 6.1 车牌定位评价

1. **IoU**  
   预测车牌框与人工车牌框的交并比：

   ```text
   IoU = area(pred ∩ gt) / area(pred ∪ gt)
   ```

   默认 `IoU >= 0.5` 判定为定位成功；可额外统计 `IoU >= 0.7` 的高质量定位成功率。

2. **中心点误差**  
   预测框中心与真实框中心的欧氏距离：

   ```text
   center_error = sqrt((cx_pred - cx_gt)^2 + (cy_pred - cy_gt)^2)
   ```

3. **定位成功率**

   ```text
   plate_success_rate = 定位成功图像数 / 总图像数
   ```

### 6.2 字符分割评价

1. **字符数量准确率**

   ```text
   count_accuracy = 预测字符数量为 7 的图像数 / 总图像数
   ```

2. **字符框 IoU**

   若标注了字符框，则按 x 坐标排序后将预测字符框与真实字符框匹配，计算平均 IoU。

3. **边界误差**

   对每个字符框比较左右边界：

   ```text
   boundary_error = mean(abs(x_left_pred - x_left_gt) + abs(x_right_pred - x_right_gt))
   ```

4. **过分割与欠分割**

   - 过分割：预测字符数量大于 7。
   - 欠分割：预测字符数量小于 7。
   - 统计每种算法的过分割次数和欠分割次数。

### 6.3 端到端评价

端到端成功定义为：

1. 车牌定位成功，即 `IoU >= 0.5`。
2. 字符数量预测为 7。
3. 字符框平均 IoU 达到设定阈值，即 `char_mean_iou >= 0.45`。该阈值考虑到人工字符框存在一定主观冗余，尤其窄体字符 `1` 的人工框可能宽于实际笔画。
4. 可视化检查中字符切分线基本落在字符间隔处。

统计指标：

| 指标 | 说明 |
|---|---|
| `end_to_end_success_rate` | 端到端成功率 |
| `avg_runtime_ms` | 单张图平均运行时间 |
| `failure_plate_not_found` | 车牌未定位次数 |
| `failure_wrong_plate` | 定位到车灯、格栅等错误区域次数 |
| `failure_char_over_split` | 字符过分割次数 |
| `failure_char_under_split` | 字符欠分割次数 |
| `failure_glare` | 主要由强反光导致失败的次数 |

## 11. 预期结论方向

在最终结果未知之前，可以先设定以下分析方向：

1. **形态学方法适合车牌定位**  
   车牌字符边缘密集，通过边缘检测和横向闭运算能够把字符区域连接成候选车牌区域。

2. **颜色阈值方法对蓝牌、黄牌有效，但对白牌和强反光不稳定**  
   蓝牌、黄牌在 HSV 空间中更容易分离；白牌容易与车灯、反光、车身亮区域混淆。

3. **投影法是最稳定的基础字符分割方法**  
   在车牌 ROI 准确、二值图清楚的情况下，垂直投影能快速切分字符。

4. **连通域方法适合字符分离明显的图像**  
   但对字符断裂和粘连比较敏感，需要形态学合并或过滤规则辅助。

5. **区域增长和分水岭适合作为困难样本补充方法**  
   区域增长可改善局部断裂，分水岭可尝试处理粘连字符，但两者对初始种子、marker 和阈值较敏感。

## 12. 风险与备选方案

| 风险 | 影响 | 备选方案 |
|---|---|---|
| 人工字符框标注耗时 | 字符 IoU 难以计算 | 当前 30 张图已按 7 字符完成标注，后续直接使用字符框评价 |
| 车牌定位失败较多 | 端到端结果不稳定 | 增加人工 ROI 字符分割实验，单独评价字符分割能力 |
| 强反光导致二值化失败 | 字符前景丢失或连成一片 | 对强反光样本单独分类分析，使用自适应阈值和局部增强 |
| 白牌颜色阈值误检 | 车灯或反光区域被选中 | 白牌定位必须结合长宽比、边缘密度和位置先验 |
| 区域增长扩散 | 前景区域过大 | 设置面积上限、边界限制和更严格相似性阈值 |
| 分水岭过分割 | 字符被切碎 | 使用标记控制，并将 marker 数量与预期字符数结合 |
| 时间不足 | 方法过多无法全部完善 | 优先完成“边缘/形态学定位 + 阈值/投影字符分割 + 统一评价可视化” |
